데이터의 검색


Oracle Full-table scan
처음부터 하나하나 검색해야 한다.

Oracle Index access
인덱스를 통해 검색한다.

이런 상황에서는 인덱스를 만드는게 좋습니다.

  • 인덱스화 된 컬럼이 자주 쿼리되고 테이블 전체의 작은 부분(2~4%)을 반환할 때
  • 컬럼 또는 인덱스화 된 컬럼에 Referential integrity constraint가 걸려 있을 때
  • Unique key constraint가 테이블에 걸려있고 이를 인덱스화 할 때
