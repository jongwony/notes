INDEX 생성 지침

  • 인덱스화 된 컬럼이 자주 쿼리되고 테이블 전체의 작은 부분(2~4%)을 반환할 때

  • 컬럼 또는 인덱스화 된 컬럼에 Referential integrity constraint가 걸려 있을 때

  • Unique key constraint가 테이블에 걸려있고 이를 인덱스화 할 때

Index의 종류

  • Bitmap Indexes

  • Function-Based Indexes

index scan의 종류

  • Index Full Scan

  • Index Fast Full Scan

  • Index Unique Scan

왜 Index가 사용되지 않았나

  • Optimizer가 인덱스를 사용하기에 효율적이지 않다고 판단할 경우

  • Function Based Index를 사용하지 않고 인덱스 컬럼을 조건절에서 가공할 경우

  • is (not) null 조건이나 < > 비교를 사용할 경우

  • Column에 산술 연산자(mathematical operation)를 사용했을 경우

  • Column에 연결 연산자(concatenate)를 사용했을 경우

  • 데이터 타입의 묵시적(implicit) 형변환

예외가 있을 수 있고, 정상적으로 인덱스 사용이 불가능할 뿐
사용자체가 불가능한 것은 아니다.
